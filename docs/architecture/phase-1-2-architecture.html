Scout Phase 1 + 2 Architecture

Phase 1 remains the deterministic trail safety black box. Phase 2 reads persisted evidence into a file-backed Brain for audit, replay, remote status, and bounded decision support. Phase 3 will operationalize the bridge without letting Phase 2 control emergency escalation.

Evidence Boundary

Allowed now: manual import of persisted Phase 1 incident package JSON.

Rejected: Phase 2 inside /safety/*, /pdr/update, risk rules, map checks, or escalation.

Ownership Boundary

Concern Owner Rule
Emergency escalation Phase 1 Phase 2 data cannot decide or downgrade L3/L4 behavior.
Persisted incident evidence Phase 1 produces, Phase 2 reads Phase 2 must not rewrite Phase 1 packages.
Automatic Brain writes Phase 2 Allowed only for deterministic facts and measurements.
Model interpretation Phase 2 Append-only, provenance-required, never an observed fact.
